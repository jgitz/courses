 .... to modify the table declaration configuration..... we use 2 lines of statements 


 - TO ALTER the DATATYPE of COLUMN in POSTGRES: 

      :: ALTER TABLE <table_name>
      :: ALTER COLUMN <column_name> SET DATA TYPE <NEW_DATA_TYPE>;

          ** SEMI COLUMN ; is on the SECOND LINE

- TO RENAME the COLUMN NAME in POSTGRES:

     :: ALTER TABLE <table_name>
     :: RENAME COLUMN <old_column_name> TO <new_column_name>;

          ** same in mysql

- TO set NOT NULL constraint to column in POSTGRES:

     :: ALTER TABLE <table_name>
     :: ALTER COLUMN <column_name_1> SET NOT NULL,
     :: ALTER COLUMN <column_name_2> SET NULL ;

          ** multiple ALTER statements in postgres  inside the same table can be seperated by COMMA ,   

- to UPDATE a TABLE RECORD with DATA VALUES 

     :: UPDATE <table_name>
     :: SET <column_name_1> = <data values>
     :: WHERE <column_name_2> = <data value>;

          ** same in MYSQL  

- to alter a table column with a CHECK CONSTRAINT in POSTGRES   

     :: ALTER TABLE <table_name>
     :: ADD CONSTRAINT <custom_constraint_name> CHECK (**eg condition** <column_name> > 0);

          ** same for MYSQL

          ** here no COLUMN keyword is used as column name is used inside the CHECK condition

          ***** before altering the table..... UPDATE the table RECORD DATA VALUES to OBEY this condition using UPDATE COMMAND.... only after that use this command.... else error
