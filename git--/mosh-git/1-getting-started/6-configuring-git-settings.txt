git settings can be applied to 3 different levels
    1) SYSTEM LEVEL : settings get applied to all users of the device

    2) GLOBAL LEVEL : settings get applied to all repositories of the current user

    3) LOCAL LEVEL : settings get applied only to the current repository

git configuring

    - to make a FOLDER into a GIT REPOSITORY by initializing git: 

        - open terminal and cd into the folder 

        - use command 

            :: git init :: 

            ** now a .git HIDDEN FOLDER will appear when the command is a success 

    - to set name GLOBALLY (i.e to all future repos made in the device)
        - :: git config --global user.name "<name>" ::

    - to set email GLOBALLY ((i.e to all future repos made in the device))
        - :: git config --global user.email <email id> :: 

        **** no double "" for email id

    - to set vscode as default editor... goto git bash and type
        - :: git config --global core.editor "code --wait" ::
        
        - to open the vscode global settings text file
            - :: git config --global -e ::

        - to handle end of line....
            - for windows 
                :: git config --global core.autocrlf true ::

            - for mac/ linux
                :: git config --global core.autocrlf input ::

** extra 

    - to add a specific username and email to a repository.... such that that username and email will only be used in that repository... use 

        :: git config --local user.name "<user name>"

        :: git config --local user.email "<email>"

        ** shorthand 

            :: git config user.name "<user name>"

            :: git config user.email "<email>"


    - to view current user and user email

        :: git config --get user.name 

        :: git config -- get user.email 

         ** or use shorthand 

            - :: git config user.name :: to display username

            - :: git config user.email :: to display user email id

    - to view ALL USER CONFIG DETAILS... use 

        - :: git config -l :: 

            ** to quit from the output line press 'q' on the keyboard
